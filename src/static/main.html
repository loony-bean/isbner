About

For decades, ISBN served as unique book identifier. Nowadays searching Web for books or creating references for your library, rarely you will find a software program or Web service that does not search for book information by ISBN code.

In the Web there are many providers of such information. Each of them uses it's own manner of presentation and own data model. It can even happen that some piece of basic information about a book (like title or publication date) can have different values obtained from different sources. Also, not all Web sources provide a way to obtain information programmatically, but using Isbner API it is possible to query the data they store.

This service utilizes the idea of aggregated search. This means that the information from different sources is analized, weighted and aggregated into unified data package. This allows Isbner to provide users with more qualitative search results.

The other idea behind the project is to propagate semantic markup for embedding book information within Web documents. There is a variety of formats available, and no one seems to be dominating now. You can see an example of how Isbner can present book information in the table below.

  • Schema
  • RDFa
  • Microformats
